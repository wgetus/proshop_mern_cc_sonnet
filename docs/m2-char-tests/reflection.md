Characterization-тесты показали, что у `orderDetailsReducer` маленький внешний контракт, но в нём есть несколько странных поведений, которые рефактор обязан сохранить. Неизвестный action возвращает ровно тот же объект state, success-action без проверок кладёт любой payload в `order`, fail-action без проверок кладёт любой payload в `error`, а отсутствующий `action` по-прежнему падает при чтении `action.type`. Поэтому рефактор поменял только форму кода: `switch` заменён на map обработчиков и маленькие pure-функции, а все возвращаемые значения остались прежними.

Первый запуск frontend-тестов упал ещё до выполнения Jest-тестов: preflight-проверка Create React App нашла несовместимый hoisted `babel-jest@30.3.0`, хотя `react-scripts@3.4.3` ожидает `babel-jest@^24.9.0`. Запуск с `SKIP_PREFLIGHT_CHECK=true` обошёл эту проблему окружения, и characterization suite прошёл. Первая попытка запустить скопированный тест из `docs/m2-char-tests` тоже упала, потому что root Jest воспринял `frontend/src/constants/orderConstants.js` как CommonJS и не смог распарсить ESM `export`; после того как копии эксперимента стали self-contained и получили локальные action type constants, эта схема тоже стала runnable и прошла.
